在 TDD 的“红-绿-重构”循环中：
- 红灯（Red）： 你先写一个单元测试，它描述了你想要实现的新功能，但因为这个功能尚未实现，所以这个测试会失败（Failing）。
- 绿灯（Green）： 此时，第一条规则（你不得编写任何产品代码，除非是为了使一个失败的单元测试通过）就生效了。
    - 你现在编写最少量的产品代码。
    - 你的目标就是让刚才那个处于失败状态的测试用例运行成功（Passing）。
- 重构（Refactor）： 一旦所有测试（包括你刚刚通过的那个）都通过了（Passing/Green），你就可以安全地重构代码。

# 支付
## 二维码订单
- 系统为每笔订单生成一个唯一的二维码，作为该订单的身份凭证。用户或商家可以通过扫描这个二维码，快速获取订单信息、完成支付、验证身份或查询状态。
## 使用银行卡（信用卡/借记卡）创建支付订单
- 使用银行卡（信用卡/借记卡）创建支付订单：指用户在支付时选择银行卡支付，系统会生成一个与该卡关联的支付订单。
## 使用银行票据（如支票、汇票）作为支付手段创建订单
- 使用银行票据（如支票、汇票）作为支付手段创建订单：在企业或大额交易中，可能使用银行支票、本票、汇票等票据支付，系统会生成一个订单，标记为“票据支付”，并等待票据到账或承兑。
- 截至2025年，Pix（二维码即时支付）已经成为巴西最普遍、最受欢迎的支付方式，在日常消费、网购、转账等场景中广泛使用。
## 支付方式使用
** 小样 **
- 二维码支付
** 批量购买 **
- 银行票据
- 银行票据支付（如银行本票、支票、汇票）在实际使用中确实存在多项限制，主要包括以下几个方面：
    - 1. 金额限制
        单张票据金额有上限：
            如电子商业汇票单张金额最高为 10亿元人民币；
            银行承兑汇票单张限额通常为 1000万元人民币（如工商银行）；
            支票理论上无上限，但受限于票据栏位，最大金额为 9,999,999,999.99 元（差1分为10亿元）。
        银行可能设定最低金额限制，尤其是本票或汇票，通常用于大额交易
    -  2. 使用区域限制
        银行本票通常只能在同一票据交换区域内使用，如一个城市或经济区，跨区域使用可能不被接受
    - 3. 兑付期限限制
        商业汇票付款期限最长不得超过6个月；
        支票通常要求在出票日起10天内提示付款；
        银行本票也需在规定期限内到指定银行兑付，逾期可能无法兑付或需额外手续
    - 4. 使用场景限制
        银行票据多用于企业间大额交易、资金调拨，个人日常消费使用较少。
        某些票据（如汇票）要求有真实交易背景，不得用于虚假交易或融资套利
    - 5. 风险控制与额度管理
        银行对票据业务设有风险控制指标：
        如银行承兑汇票余额不得超过银行总资产的15%；
        保证金余额不得超过存款规模的10%；
        企业申请开票时，需有授信额度或缴纳保证金，否则无法开票
- 银行票据的类型：
    - **支票**
    - 定义：支票就是一张由银行发行的、你亲手填写的“付款指令”或“取款条”。
    - 出票人：企业或个人
    - 付款人：开户银行
    - 使用方式：出票人填写金额、收款人、日期并签章，收款人背书后去银行兑现或转账
    - 适用范围：同城日常结算，如房租、工资、小额采购
    - 优点:使用灵活、可背书转让、零手续费
    - 缺点：易出空头支票、信用依赖出票人、仅限同城，跨国支票成本高昂
    - 为什么在支票上背书：
        - 1. 委托收款（最常用）
            场景： 您收到一张支票，要把它存入自己的银行账户。
            操作： 您在支票背面指定区域签上自己的名字（或盖上预留银行印鉴），然后交给银行。
            含义： 这相当于您对银行说：“我是这张支票的合法收款人，现在我委托你们银行帮我把这笔钱收进来，存到我的账户里。”
            这是个人去银行存支票时必须要做的一步。
        - 2. 转让给他人（转让权利）
            场景： 您收到一张支票，但想用它来支付给您的供应商或朋友。
            操作： 您在支票背面写上“支付给XXX（被背书人名称）”，然后在下面签上您自己的名字（背书人）。
            含义： 这相当于您说：“我把这张支票的收款权利，转让给XXX。” 对方（XXX）就可以拿着这张支票去银行取现或入账了。
            这种背书可以连续进行，形成一条转让链：A → B → C。
    - **银行本票**
    - 定义：银行本票就是银行自己承诺无条件付款的票据。
    - 出票人：银行
    - 付款人：银行
    - 使用方式：申请人先存款，银行签发本票，收款人见票即付
    - 适用范围：同城大额交易，如房产、车辆、保证金
    - 优点：见票即付、信用高、无空头风险
    - 缺点：需先存款、占用资金、仅限同城

    - **跨国银行本票**
    - 定义：跨国银行本票是由一家境外银行签发，承诺在其境外分支机构或代理行见票即付确定金额给收款人或持票人的票据。
    - 缺点：
        1. 极高的伪造风险：是金融诈骗的重灾区。
        2. 处理速度极慢：光票托收流程长达1-3个月。
        3. 费用高昂：承担高额的托收手续费、邮费和可能的中间行费。
        4. 资金周转效率极低：长时间占用资金额度。
        5. 手续繁琐：必须亲自到银行办理托收。

    - **银行汇票**
    - 定义：银行汇票是银行根据申请人要求，签发给其在外地进行转账或取现的票据。
    - 出票人：银行
    - 付款人：银行
    - 使用方式：银行签发后交申请人，收款人凭票到指定银行兑付
    - 适用范围：异地结算，如跨省采购、工程款
    - 优点：可异地使用、银行信用、流通性强	
    - 缺点: 需支付手续费、到账略慢于Pix

    - **电汇**
    - 定义：电汇就是汇款人将款项交给银行，银行通过电报、电传或SWIFT等电子化方式，将款项汇给指定收款人所在银行的一种汇款方式。银行系统内部的“电子化加急快递”，专门用于“快递”资金。汇款人已经提供了收款人的准确银行账号。此时，资金会自动存入该账户。
    - 
    - **商业汇票**
    - 定义：商业汇票是一种由企业或个人签发，委托或承诺在指定日期无条件支付确定金额给收款人的票据，它是一种“企业打的欠条”或“延期付款承诺”。
    - 企业（通常是卖方或供应商）
    - 付款人：企业（买方）或银行（如果银行承兑
    - 使用方式：企业签发后由付款人（或银行）承兑，到期兑付
    - 适用范围：企业间赊销、供应链金融，可延期6个月
